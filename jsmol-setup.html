JSmol Setup Instructions

The Lewis Structure Generator's "View in 3D" feature uses JSmol, a JavaScript-based molecular viewer from the Jmol project. Because JSmol is ~15 MB and we don't want to bloat the repo by checking it into git, you'll install it once manually and commit the files.

Before you start: This needs to be done on a computer (not a phone), and you'll need about 15 MB of free space in the repo. You'll do this once; after that, the 3D viewer just works.

What you'll end up with

Your repo should have this structure after setup:

chrisl18rel.github.io/ ├── index.html ├── lewis-vsepr.js ├── jsmol-adapter.js ├── ... (other JS/CSS files) ├── js/ │ ├── jquery/ │ │ └── jquery.min.js │ └── JSmol/ │ ├── JSmol.min.nojq.js │ └── j2s/ │ └── (hundreds of files — JSmol's renderer assets) └── docs/ └── jsmol-setup.html (this file)

Step 1 — Download jQuery

1 Go to jquery.com/download/ and click "Download the compressed, production jQuery" (the file is named something like jquery-3.7.1.min.js).

2 Create a folder called js/jquery/ inside your repo.

3 Save the downloaded file as js/jquery/jquery.min.js (rename it if needed — our index.html looks for exactly that filename).

Step 2 — Download JSmol

1 Go to the Jmol SourceForge page: sourceforge.net/projects/jmol/files/Jmol/

2 Navigate to the newest Version X.X.X folder, then inside to the folder for that version's binaries. You want the file named Jmol-X.X.X-binary.zip (around 90–100 MB).

3 Download and unzip it. Inside, find the folder named jsmol/. This is the one you need — the other folders are for the desktop Jmol application, which we don't need.

4 Create a folder called js/JSmol/ inside your repo.

5 Copy the contents of the extracted jsmol/ folder (not the folder itself) into your repo's js/JSmol/. You should end up with js/JSmol/JSmol.min.nojq.js directly and a js/JSmol/j2s/ subfolder with hundreds of files.

Tip: The jsmol/ folder contains a lot of example HTML files and unused variants of JSmol. You can delete everything except these to save space:
  • JSmol.min.nojq.js (the minified, no-jQuery-bundled version — we load jQuery separately)
  • j2s/ (entire folder — these are the renderer assets)
You can safely delete: the HTML example files, php/, idioma/, and the other .js variants of JSmol. This brings the size down from ~90 MB to ~15 MB.

Step 3 — Commit and push

1 In your terminal:

git add js/
git commit -m "Add JSmol library for 3D molecule viewer"
git push

2 Wait a minute for GitHub Pages to redeploy.

3 Open your site. In the Lewis tool, generate a structure like phenol or alpha-glucose. The "View in 3D" button below the Lewis canvas should now be enabled.

Done (basic install)! Click "View in 3D" and JSmol will fetch a 3D model from CACTVS (NIH's chemical structure service) and render it below the Lewis structure. Drag with your mouse to rotate; scroll to zoom.

Step 4 — Build the 3D cache (optional but recommended)

By default, every "View in 3D" click hits NIH's CACTVS server to fetch the 3D structure. That's about 1–5 seconds per load, and it doesn't work offline. The tool includes a build script that pre-fetches 3D structures for every supported molecule and saves them to your repo, so loads are instant and work even when the classroom internet is flaky.

These instructions assume Mac. You'll need Node.js installed, your repo cloned locally (via Claude Cowork or git clone), and Terminal access. If you've already set up clasp for your Apps Script projects, you already have everything you need.

Step 4a — Install Node.js (if you haven't already)

1 Open Terminal (press Cmd + Space, type "Terminal", press Return).

2 Check if Node is already installed by running:

node --version

If you see something like v20.10.0 (any v18 or newer), you're good — skip to Step 4b. If you get "command not found" or a version older than v18, install Node from nodejs.org — grab the LTS version, run the installer, and then re-open Terminal and re-check with node --version.

Step 4b — Make sure your repo is cloned locally

You need the files on your computer, not just on GitHub. If you already have the repo on your Mac (from Claude Cowork or a previous git clone), you're set. Otherwise, in Terminal:

cd ~/Documents
git clone https://github.com/chrisl18rel/chrisl18rel.github.io.git

This downloads the repo into ~/Documents/chrisl18rel.github.io/. If you cloned it to a different folder, substitute that path throughout the remaining steps.

Step 4c — Navigate to the repo

1 In Terminal:

cd ~/Documents/chrisl18rel.github.io

2 Verify you're in the right folder by listing what's there:

ls

You should see files like index.html, lewis-vsepr.js, jsmol-adapter.js, scripts/, etc. If not, you're in the wrong folder — use cd to navigate to wherever your repo actually is.

Step 4d — Run the build script

1 From the repo root, run:

node scripts/build-3d-cache.js

2 You'll see output like this as it works:

Building 3D cache for Lewis Structure Generator
================================================

Will fetch 98 unique molecules from CACTVS
Each request is paced at 400ms to be polite to the server.
Expected runtime: ~40s

[1/98] 1,2-dibromobenzene … OK (2341 bytes)
[2/98] 1,2-dichlorobenzene … OK (2248 bytes)
[3/98] 1,2-dinitrobenzene … OK (2583 bytes)
...

3 The script takes about 45–90 seconds. It fetches one molecule every 400ms (pacing is on purpose — we're polite to NIH's free service).

4 When it finishes, you'll see:

================================================
Done. 94 cached, 4 failed.

Failures (these will still work via live CACTVS at runtime):
  - sulfur hypofluorite: HTTP 404
  - trithiophosphite: HTTP 404
  ...

Failed molecules aren't blocking — the tool falls back to live CACTVS for those. If you want, share the failure list with me and I'll figure out alternate CACTVS names we can use.

Tip: The script is resumable. If you interrupt it (Ctrl+C) or your internet drops, run it again and it'll skip molecules that are already cached.

Step 4e — Commit and push the cache

1 Still in Terminal, from the repo root:

git add js/3d-cache/
git commit -m "Build 3D structure cache"
git push

2 Wait ~30 seconds for GitHub Pages to redeploy, then test your site. Click "View in 3D" on a common molecule like water or benzene — it should load instantly now (instead of the 1–5 second CACTVS fetch).

Done (with cache)! 3D models for the cached molecules now load instantly and work offline. Uncached molecules still fall back to live CACTVS.
When to rebuild the cache: Re-run the build script if you add new molecules to the name table (in jsmol-adapter.js's NAME_MAP) or if CACTVS updates their 3D models and you want the fresh versions. The process is the same: cd into the repo, run node scripts/build-3d-cache.js, commit and push.

Step 5 — Build the IUPAC name cache (optional but recommended)

Just like the 3D cache, IUPAC names are fetched from CACTVS. Without a local cache, every molecule shown in the tool triggers a CACTVS request to get its IUPAC name. With a local cache, names display instantly and work offline.

1 From the repo root (where you ran the 3D cache script):

node scripts/build-iupac-cache.js

2 Same as the 3D cache: the script paces requests at 400ms and takes about 45–90 seconds. Output looks like:

[1/98] 1,2-dibromobenzene … OK → 1,2-dibromobenzene
[2/98] 1,2-dichlorobenzene … OK → 1,2-dichlorobenzene
...
[12/98] alpha-d-glucose … OK → (2R,3R,4S,5S,6R)-6-(hydroxymethyl)oxane-2,3,4,5-tetrol
...

3 When finished, commit and push:

git add js/iupac-cache/
git commit -m "Build IUPAC name cache"
git push
Done (with name cache)! The tool now shows IUPAC names instantly under the Lewis structure. Users can also type IUPAC names in the input box (e.g., "propan-2-ol" or "acetic acid") and the tool will look up the formula via CACTVS and render the structure.
Heads up: IUPAC name input always goes through live CACTVS (we can't pre-cache unknown user inputs). So that feature needs a working internet connection. The cached names are only for the display under the Lewis canvas.

Troubleshooting

The button stays gray/disabled saying "library not installed"

JSmol didn't load. Open your browser's DevTools (F12) → Console tab and look for 404 errors on JSmol.min.nojq.js or jquery.min.js. The paths in index.html are js/JSmol/JSmol.min.nojq.js and js/jquery/jquery.min.js — double-check your folder structure matches exactly.

The button is enabled but the 3D view says "loading…" forever

CACTVS (the NIH server that provides 3D structures) might be slow or down. Check your internet connection, try a different molecule, or try again in a few minutes.

The 3D view shows for some molecules but not others

CACTVS doesn't have every molecule. Common ones (water, methane, benzene, glucose) work reliably. Unusual ones (like made-up polyatomic ions) may not. This is expected behavior.

GitHub rejects my push — file too large

GitHub blocks files over 100 MB. JSmol's individual files are all small, but if you accidentally committed the full SourceForge zip (~90 MB) or the Java application JAR, that might be the culprit. Make sure you only commit the contents of the jsmol/ folder.

Why this setup?

JSmol is self-contained — once vendored, it runs entirely in the browser with no server-side component. Your GitHub Pages site serves the JS files; each visitor's browser runs the viewer. The only external call is to CACTVS (cactus.nci.nih.gov) to fetch 3D coordinates for named molecules. Nothing comes back to your machine.

If CACTVS ever goes down or becomes unreliable, a future update (R7-2) will pre-fetch 3D models for common molecules and ship them alongside the app, so the most common use cases keep working offline.
